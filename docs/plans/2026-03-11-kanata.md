# Kanata Cross-Platform Coverage Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Install Kanata on Windows and Linux with `user_setup`, `not work`, `not ephemeral` guards, using a Home Manager user service on Nix Linux, a system package + user autostart on Chimera, and Scoop + autostart on Windows.

**Architecture:** Add a small Home Manager module to install `pkgs.kanata` and define a systemd user service with a `ConditionPathExists` on the config file. Wire it from `home.nix.tmpl` under the specified guards. For Chimera, install the system package and configure user autostart using the native service manager (likely `dinitctl --user`), with XDG autostart as fallback. For Windows, install via Scoop and add explicit autostart in the login-items script if Kanata does not self-register on first run.

**Tech Stack:** Chezmoi templates, Nix Home Manager, Scoop, Chimera apk, PowerShell.
